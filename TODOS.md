## P2 — Before public launch (5+ practices)

### T1: Migrate staff_* role pattern to practice_id + role_level

**What:** Replace `staff_brentwood`, `staff_lewisburg`, `staff_nashville` etc with a two-field model: `role = 'staff'` + `practice_id` (already on user_profiles). Update auth middleware to check practice_id instead of parsing the role string.

**Why:** Every new location_code mints a new hardcoded role name. The current pattern doesn't scale and makes authorization logic fragile as practices grow.

**Where to start:** `artifacts/api-server/src/middleware/auth.ts` (StaffRole type), `lib/db/src/schema/user_profiles.ts`, all `role.startsWith("staff_")` checks in routes.

**Effort:** L human (~2 days) / CC ~1 hour. Touches user_profiles data, auth middleware, and all role checks across frontend and backend.

**Priority:** P2 — not urgent at 3 practices, critical before 10+.

**Depends on:** 5+ active practices (so the migration is worth the coordination cost).

### T2: Per-practice referral counter and billing dashboard

**What:** A super_admin view (new tab or section in `/dashboard` or `/offices`) showing referral count and estimated revenue per office. Query from `referral_events` by `office_id`. Export as CSV for invoicing.

**Why:** Manual counting of referrals per practice works for 1-2 practices. The data is already in `referral_events` — this is just a GROUP BY query + UI.

**Where to start:** `artifacts/api-server/src/routes/dashboard.ts` (add per-office breakdown), `artifacts/rippl/src/pages/offices.tsx` or a new `/billing` page.

**Effort:** M human (~4 hours) / CC ~20 min.

**Priority:** P2 — build after 3+ active practices.
